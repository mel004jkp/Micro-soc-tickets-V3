Este documento detalla cada uno de los cambios, códigos, y configuraciones que han

--------------------------------------------------------------------------------
1. INTEGRACIÓN CON FORMSPREE (NOTIFICACIÓN DE TICKETS POR CORREO DIRECTO)
--------------------------------------------------------------------------------
Se configuró una conexión AJAX en tiempo real para despachar los datos de los 
tickets ingresados por tus clientes directamente a tu manejador de correos por medio 
del endpoint provisto:
 - Endpoint: https://formspree.io/f/mnjylqbd
 - Archivo afectado: `/src/components/ClientHome.tsx`

Código representativo integrado para envío de notificaciones:
```typescript
try {
  const response = await fetch("https://formspree.io/f/mnjylqbd", {
    method: "POST",
    headers: {
      "Accept": "application/json",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      ticket_id: uniqueId,
      numero: nextNum,
      email: user.email,
      nombre_solicitante: user.nombre,
      empresa: user.empresa || 'PYME',
      incidente: incidente,
      prioridad: prioridad,
      observaciones: observaciones.trim(),
      fecha_creacion: newTicket.fechaCreacion,
      sistema: "Micro-SOC Cybersecurity Platform Alerts"
    })
  });
  if (response.ok) {
    formspreeNotified = true;
  }
} catch (fsErr) {
  console.warn('Failed to send email alert via Formspree:', fsErr);
}
```

--------------------------------------------------------------------------------
2. REMOCIÓN DEL BOTÓN/RECUADRO DE AUTOREGISTRO DE CLIENTES
--------------------------------------------------------------------------------
Para evitar desajustes o "lios" en las tablas de Supabase, se eliminó por completo 
el formulario interactivo de registro automático ("Registrarse Nuevo Cliente"). 
A partir de ahora, tú como administrador puedes crear directamente los usuarios de 
forma limpia en tu base de datos central de Supabase (auth.users / tablas de perfil).
 - Archivo afectado: `/src/components/LoginRegister.tsx`
 - Cambio en el Layout: Se alteró el diseño de 2 columnas (grid-cols-2) a un panel 
   centrado de un único módulo (flex justify-center) para hacer la interfaz del 
   portal de login limpia, moderna y profesional.

--------------------------------------------------------------------------------

--------------------------------------------------------------------------------
5. ARQUITECTURA DE DATOS RECOMENDADA EN SUPABASE (TUS TABLAS)
--------------------------------------------------------------------------------
Para evitar confusión con tus tablas debido a que tienes múltiples estructuras, te 
especificamos los campos requeridos para las dos interacciones principales con Supabase:

A) Para los TICKETS (Tabla recomendada: `tickets`):
   - `id` (text / uuid, Primary Key)
   - `numero` (int / numeric)
   - `cliente_email` (text / varchar)
   - `cliente_nombre` (text / varchar)
   - `incidente` (text)
   - `prioridad` (text / varchar)
   - `observaciones` (text)
   - `estado` (text / varchar, por ej: 'Abierto')
   - `fecha_creacion` (text o timestamp)
   - `respuesta_auditor` (text, opcional, por defecto '')

B) Para el BUZÓN DE QUEJAS Y SUGERENCIAS (Tabla recomendada: `suggestions`):
   - `id` (uuid, Primary Key o auto-incrementable)
   - `email` (text / varchar)
   - `empresa` (text / varchar)
   - `tipo` (text / varchar, ej: 'Queja' o 'Sugerencia')
   - `mensaje` (text)
   - `fecha_creacion` (text o timestamp)
